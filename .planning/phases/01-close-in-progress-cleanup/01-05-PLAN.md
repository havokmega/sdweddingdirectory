    will not accidentally rewrite folder paths, comments mentioning the old plugin name, or
    URLs. The only cases to be careful of are:
      * Inside-string occurrences in comments (e.g., "this hooks the legacy sdweddingdirectory plugin")
        — these are non-functional but a sed will rewrite them. That's acceptable; the resulting
        comments still parse and may even be improved (now refer to the current naming).
      * Hook names like `'sdweddingdirectory/post-cap/vendor'` (slash-namespaced filter hooks
        per CONVENTIONS.md §"Hook / Action / Filter Naming"). These contain the slash so they
        are NOT matched by `'sdweddingdirectory'` followed by closing single-quote. They would
        be matched by `'sdweddingdirectory` alone, but the sweep targets the FULL quoted literal
        `'sdweddingdirectory'` (with both quotes), so hook names are safe.
